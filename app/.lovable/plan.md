
# 民宿 AI 内容增长工具 —— MVP 原型规划

一个中文、移动端优先的轻量内容创作工作台。核心动作只有一条：**粘贴好评 → 生成笔记 → 分块复制发布**。第一版全部使用模拟数据，不接真实 AI、数据库、登录、支付。

---

## 1. 信息架构

```text
应用根
├─ 首页 / 工作台          （核心：填资料 + 粘好评 + 生成）
│   ├─ 民宿资料卡片        （最小字段，可展开补充）
│   ├─ 好评输入区
│   ├─ 每日额度指示
│   └─ 生成按钮
├─ 生成结果页              （标题 / 正文 / 标签 / 配图 / 依据 / 风险）
├─ 历史记录页              （本地保存，最近 20 条）
├─ 历史详情                （复用结果页组件）
└─ 使用说明 / 内容原则     （静态页，讲清"不是游客视角"）
```

底部固定 Tab 导航（移动端优先）：**工作台 / 历史 / 说明**。

---

## 2. 页面清单

| 路由 | 名称 | 作用 |
|---|---|---|
| `/` | 工作台 | 填资料 + 粘好评 + 触发生成 |
| `/result` | 生成结果 | 展示笔记草稿，分块复制 |
| `/history` | 历史记录 | 本地保存的历次生成 |
| `/history/$id` | 历史详情 | 查看/复制某次结果 |
| `/guide` | 使用说明与内容原则 | 视角边界、避免踩红线 |

---

## 3. 完整用户流程

```text
首次进入 /
  │
  ├─ 顶部：欢迎条 + 今日额度 5/5
  ├─ 民宿资料卡片（空）——「先填一下你的民宿，只要 2 项」
  │     必填：民宿名、所在城市
  │     可选：房型数、卖点标签、周边关键词、宠物政策等（折叠）
  │
  ├─ 好评输入框（大 textarea，占屏幕主要位置）
  │     placeholder：真实好评示例
  │     字数计数：建议 50–800 字
  │
  └─ 底部主按钮：「生成小红书笔记」
         │
         ▼
   点击生成
         │
    ┌────┴─────┐
    │ 校验失败  │ → 内联红字，例：好评太短 / 民宿名未填 → 不扣额度
    │ 校验通过  │ → 进入加载态（1.5–2.5s 模拟）
    └────┬─────┘
         ▼
    模拟 AI 返回
         │
    ┌────┴──────────────┐
    │ 成功              │ → 跳 /result，额度 -1，写入本地历史
    │ 模拟失败（小概率）│ → 提示"生成失败，请重试"，不扣额度
    │ 额度已用完        │ → 按钮变灰 + 说明「明天 0 点重置」
    └────┬──────────────┘
         ▼
   /result 结果页
     ├─ 3–5 个标题（点击复制单条，可切换主选）
     ├─ 正文（整块复制）
     ├─ 8–15 个话题标签（点单个复制 / 复制全部）
     ├─ 3–6 条配图建议（文字描述，不生成图）
     ├─ 生成依据摘要（"基于你提供的这几句好评…"）
     ├─ 风险提醒（未提供的信息、避免游客视角、极限词检查）
     └─ 底部操作栏：
          [复制整篇笔记]  [重新生成（-1 次）]  [返回修改]
```

复制整篇 = 主选标题 + 正文 + 话题标签，按小红书发布习惯拼接。

## 4. 每页关键 UI 组件

### 4.1 工作台 `/`
- `QuotaBadge`：今日剩余 X/5，用小圆点或进度条
- `PropertyCard`：民宿资料，默认收起可选字段，改动即本地保存
- `ReviewTextarea`：大输入框 + 字数计数 + 清空按钮
- `TipsRow`：一行小字「使用民宿主视角 · 不编造事实」
- `GenerateButton`：主 CTA，禁用态显示原因

### 4.2 生成结果 `/result`
- `TitlePicker`：标题列表，单选一个作为主标题，每条右侧「复制」
- `BodyBlock`：正文卡片 + 右上角「复制正文」
- `TagChips`：标签胶囊，点击单个复制，顶部「复制全部标签」
- `ImageIdeaList`：配图建议编号列表（纯文字）
- `RationaleAccordion`：生成依据（折叠展开）
- `RiskCallout`：风险提醒卡片（琥珀色，非红色，语气克制）
- `StickyActionBar`：底部固定 —— 复制整篇 / 重新生成 / 返回

### 4.3 历史 `/history`
- `HistoryList`：卡片列表，显示民宿名、时间、主标题预览
- 空态：插画 + 「还没有生成记录」
- 每条右侧「查看」「删除」

### 4.4 说明 `/guide`
- 三块内容：**我们是谁的视角** / **不会写什么** / **风险提醒如何看**

## 5. 状态设计

| 状态 | 出现位置 | 表现 |
|---|---|---|
| **空态** | 工作台首次访问 | 资料卡引导「先填 2 项」；好评框显示示例 placeholder |
| **空态** | 历史页 | 插画 + 引导「去生成第一篇」 |
| **加载中** | 点击生成后 | 主按钮变「正在生成…」+ 骨架屏结果卡 + 温和文案「正在整理住客的话…」 |
| **成功** | /result | 顶部轻提示「已生成 · 剩余 4/5」，2 秒消失 |
| **复制成功** | 任意复制按钮 | Toast「已复制」 |
| **校验失败** | 工作台 | 内联红字，字段边框高亮，不扣额度 |
| **生成失败（模拟）** | 工作台 | 顶部横幅「生成失败，请重试。本次不计入额度。」 |
| **额度用完** | 工作台 & 结果页重新生成按钮 | 按钮禁用 + 说明「今日 5 次已用完，明天 0 点自动重置」；界面其他部分仍可浏览 |

## 6. 模拟数据范围（第一版）

全部本地，无网络请求：

- **民宿资料**：`localStorage` 存 1 份档案
- **额度**：`localStorage` 存 `{ date, used }`，跨天自动重置为 0
- **历史**：`localStorage` 存最近 20 条生成结果
- **AI 生成**：一个 `mockGenerate(review, property)` 函数
  - `setTimeout` 1.5–2.5s 模拟延迟
  - 从预置模板池中按好评关键词做简单匹配挑选
  - 准备 3–4 套完整示例（海边/山野/城市/古镇风格）覆盖不同关键词
  - 主动检测：好评里没出现的事实（早餐、价格、距离…）→ 自动写进「风险提醒」，不写进正文
- **失败**：10% 随机概率触发模拟失败，用于展示失败态
- **示例好评**：内置 3 条真实感中文好评供一键填入

## 7. 推荐构建顺序

1. **路由骨架 + 底部 Tab 导航**（工作台 / 历史 / 说明三个空壳）
2. **设计令牌与基础样式**：温暖米白背景、深墨字、克制的琥珀色作强调；中文字体栈；圆角与阴影统一
3. **工作台页布局**：资料卡 + 好评输入 + 额度徽标 + 主 CTA（先纯 UI）
4. **`localStorage` hooks**：`useProperty` / `useQuota` / `useHistory`
5. **`mockGenerate` 生成器 + 模板池 + 风险检测规则**
6. **生成结果页**：分块展示 + 分块复制 + 一键整篇复制
7. **加载态 / 失败态 / 额度用完态**
8. **历史列表 + 详情**（复用结果页组件）
9. **使用说明页**（内容原则文案）
10. **移动端细节打磨**：安全区、粘性底部操作栏、Toast、可访问性
11. **示例好评 & 首次进入引导**（onboarding 提示条，不做完整向导）

## 需要在实现阶段确认的技术选择（不改变范围）

- 路由：TanStack Start 现有结构，全部为客户端页面，无 loader/server function
- 状态：轻量本地 hook + `localStorage`，不引入 zustand
- UI：现有 shadcn 组件（Button/Card/Textarea/Badge/Accordion/Toast）
- 复制：`navigator.clipboard.writeText`，降级方案 fallback
- 无后端、无 Cloud、无鉴权
