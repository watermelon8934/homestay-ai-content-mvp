# 民宿 AI 内容增长工具

面向 5-10 间房的小体量民宿老板，帮助他们把 OTA 平台好评转化为可直接复制发布的小红书笔记。

本项目是一个独立项目，不属于也不引用其他业务系统的业务、客户、数据结构或文档。

## 项目背景

小体量民宿老板通常没有专职内容运营人员，也很难持续投入精力做小红书选题、写作、复盘和优化。产品目标是让老板只需粘贴 OTA 平台好评，系统自动生成可复制发布的小红书笔记，并在后续通过公开搜索回访和内容雷达自动优化生成效果。

长期目标是：

老板只管复制发布，系统负责写、看、学、优化。

## 产品定位

- 独立收费的 AI 内容增长产品。
- 也可作为其他民宿、文旅 SaaS 的增值功能。
- 第一阶段聚焦“OTA 好评 -> 小红书笔记生成”这一条高频、清晰、低门槛路径。

## 第一版范围

老板粘贴 OTA 好评文本后，系统生成完整小红书笔记：

- 标题
- 正文
- 话题标签
- 配图建议

第一版暂不支持截图 OCR，只支持复制文本。允许老板补充民宿名称、城市/景区、卖点、适合人群、小红书主页等资料。

## 目标用户

- 5-10 间房的小民宿老板。
- 日常经营很忙，没有内容运营团队。
- 不能依赖他们持续回填数据、分析内容或维护复杂配置。
- 核心体验应尽量接近“粘贴、生成、复制发布”。

## 内容原则

- 优先生成种草型内容。
- 建议采用“民宿主/内容运营视角 + 游客好评佐证”。
- 不伪装游客真实入住。
- 不编造具体事实。
- 未提供的信息不能写成确定事实。
- 避免过度营销、极限词、虚假承诺和平台违规风险。

## 核心能力路线

### MVP

- 粘贴 OTA 好评文本。
- 补充基础民宿资料。
- 生成小红书标题、正文、标签、配图建议。
- 每账号每天最多 5 次成功生成。
- 失败不扣额度。
- 重新生成算一次。
- 留存生成记录和用户偏好。

### 发布后回访

系统已知民宿名称、城市、生成标题、正文关键句、标签和生成时间，可通过公开搜索找到老板发布的小红书笔记，记录点赞、收藏、评论等公开互动数据，并反哺后续生成。

第一阶段可先做“发布后公开搜索回访”，不做大规模平台采集。

### 内容雷达

长期希望后台自动观察小红书民宿公开内容趋势，形成选题、标题、表达方式和标签的持续优化能力。

## 技术倾向

- 知识库：保留民宿档案、生成记录、老板偏好和历史内容。
- 数据库可考虑 PostgreSQL + pgvector。
- 向量模型可考虑 Qwen3-Embedding。
- 模型接入层必须可切换。
- 大陆生产环境可优先考虑 GLM-5.2 / Qwen 等稳定可接入模型。
- GPT 可作为精品/灰度通道，但要考虑大陆接入、合规和成本风险。
- 开源组件优先选择 Apache-2.0 / MIT 等商业友好许可证。
- 可考虑 Vercel AI SDK、LiteLLM、Promptfoo、Playwright/Crawlee、Qwen3-Embedding。
- 谨慎使用许可证不够宽松或限制 SaaS 的项目。

## 建议下一步

1. 设计第一版页面流程：民宿资料填写、好评粘贴、生成结果、复制发布。
2. 设计数据表：账号、民宿档案、生成记录、额度记录、反馈记录。
3. 搭建可切换模型接入层，先跑通一个稳定模型。
4. 用 Promptfoo 建立小样本评测集，验证“不编造、不伪装游客、种草表达”的稳定性。
5. 做一个最小可用 Web 原型，先服务单个民宿老板的真实样本。
