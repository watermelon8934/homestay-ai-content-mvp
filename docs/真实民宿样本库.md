# 真实民宿样本库

## 1. 用途

这份文档用于沉淀真实民宿的小红书内容样本，帮助我们从“盲调 prompt”转向“基于好样本学习”。

每个样本不只是收集文案，而是拆解：

- 标题怎么吸引目标客人
- 开头怎么进入住宿选择场景
- 正文如何组织卖点
- 标签如何覆盖搜索词
- 图片如何承担种草作用
- 评论区是否出现咨询信号
- 哪些表达可以复用，哪些表达有风险

## 2. 采集优先级

优先采集：

- 真实民宿自营账号
- 大理、丽江、杭州、厦门、莫干山等民宿密集城市
- 有院子、古城、亲子、情侣、宠物、景观等明确场景的民宿
- 收藏、评论或咨询明显高于同类的笔记
- 评论区有人问位置、价格、房型、交通、预订方式

暂不优先：

- 纯达人广告
- 只有情绪没有住宿信息的内容
- 大量夸张词、极限词、低价噱头
- 无法被小体量民宿老板复用的内容


### 3.1 图片拆解要求

后续样本如果有配图，必须同时记录“图文是否匹配”，不要只抄正文。

每组图片至少拆这几项：

- 首图内容：首图是否能承接标题，是否一眼看出院子、房间、景观、位置或服务细节。
- 图片能证明的事实：例如院子、窗景、房间、床位、厨房、浴室、周边、餐食、多人活动。
- 图片不能证明的事实：例如具体距离、价格、安静程度、服务承诺、是否免费、是否长期提供。
- 图文错配风险：正文写了但图片没有支撑，或图片很好看但正文没有提到。

我们的目标不是让图片“好看”就行，而是让读者刷到时能迅速完成：

```text
首图吸引点开
标题给出选择理由
正文解释为什么值得住
组图逐张证明正文里的细节
```

### 4.0 已采样本 1：大理旅居住过的 3 家民宿

样本文件：

```text
samples/xhs/20260708-dali-journey-3-homestays-001.json
```

账号类型：游客/生活方式推荐视角。

公开互动：点赞 557，收藏 508，评论 45。

这条不是民宿自营账号，不能直接照搬口吻，但对 Prompt 很有价值。

可学习点：

- 标题有强情绪和真实筛选感：“住过的 N 家 / 每一家都……”。
- 开头先交代真实场景和记录动机，不急着卖点。
- 正文按“位置 + 想念理由 + 总结适合人群”组织。
- 具体写法里允许出现边界，例如位置偏、需要开车，这会增加真实感。
- 评论区出现房价、民宿名称、亲子适配等咨询信号，说明这类结构能激发转化问题。

不能照搬的点：

- 民宿主不能写“我住过”“让我想念”。
- 单店生成不能写成多民宿合集。
- 不能把游客视角直接伪装成民宿主视角。

可转化为 Prompt 的规则：

- 把“住过的 3 家”转成“住客提到的 3 个细节”。
- 把“每一家都让我想念”转成“这些细节被记住，会让我们觉得值得”。
- 单店正文可以采用：“民宿主分享动机 -> 2-3 个真实细节 -> 适合什么行程/人群”。
- 允许写克制的边界提示，例如“更适合想慢一点住的人”，不要只写优点。

样本文件：

```text
samples/xhs/20260708-dali-favorite-homestay-002.json
```

账号类型：游客/生活方式推荐视角。

公开互动：点赞 405，收藏 241，评论 37。

这条样本的价值是“生活画面”，不是民宿主身份。

可学习点：

- 标题用“住过几家后最喜欢”建立筛选感。
- 正文不是简单列卖点，而是把房间、窗景、下午茶、躺椅、阳光、月亮等写成一天的生活动线。
- 评论区有人问月份、民宿名、房间人数，说明生活方式画面能推动实际咨询。
- 图片和文案一起服务“在这里怎么过一天”的想象。

不能照搬的点：

- 民宿主不能写“我住了四家，最喜欢这家”。
- 没有输入依据时不能套用海景、下午茶、餐厅等画面。
- “不想走”这类情绪可以学习方向，但民宿主版本要更克制。

可转化为 Prompt 的规则：

- 把“我最喜欢的一家”转成“我们想让客人记住的一天”。
- 不只写设施，要写住客会怎么使用空间。
- 图片建议按生活动线组织：房间/窗景 -> 食物/饮品 -> 公共区/周边。
- 对我们云朵情书样例，可转成：窗边、院子、外卖、小甜品、回房间后慢下来。

### 4.0.3 已采样本 3：来大理住我家吧

样本文件：

```text
samples/xhs/20260708-dali-owner-rental-003.json
```

账号类型：民宿自营/房东账号。

公开互动：点赞 295，收藏 189，评论 457。

这是目前最接近民宿主自营账号的样本。它的文案不算最美，但转化信号最强。

可学习点：

- 标题直接给行动和空房信息：“来大理住我家吧（6.1 后空）”。
- 正文按住客决策顺序写：位置、交通、房间、清洁、亲子、做饭洗衣、服务。
- 评论区大量问日期、价格、房型、距离，说明这类直接文案有强转化能力。
- 房东口吻可以直接，不一定要文艺。

不能照搬的点：

- 这条偏租房/月租，不完全等于 OTA 好评转小红书笔记。
- 卖点过密，直接套用会像房源广告。
- 日期、价格、房型、宠物、儿童用品、厨房、洗衣机等信息不能编造。

可转化为 Prompt 的规则：

- 保留自营账号的直接感，但不要写成配置清单。
- 单篇正文只写 2-3 个最关键决策点。
- 优先顺序可以是：“位置省心 -> 房间/院子体验 -> 服务细节”。
- 如果输入没有价格和日期，不要生成价格和空房信息。
- 对云朵情书样例，可转成：东门附近出行省心、院子和窗边有生活感、房间每天打扫和小甜品。

### 4.0.4 已采样本 4：大理旅居来我家

样本文件：

```text
samples/xhs/20260709-dali-owner-community-yard-004.json
```

账号类型：民宿自营/主理人账号。

公开互动：点赞 235，收藏 147，评论 320。

这条是非常有价值的民宿主口吻样本。它不是靠房间配置转化，而是靠人格、社交氛围和生活方式转化。

可学习点：

- 标题击中明确需求：“旅居来我家”“不用出去找搭子”。
- 开头像民宿主在招呼朋友，不像客服或品牌广告。
- 正文先写老板夫妻、小院氛围、咖啡、聊天、看日落、活动，再写位置和房间。
- 评论区大量问价格、月租、短租、是否有房，说明人格化内容也能带来实际咨询。

不能照搬的点：

- 如果民宿没有真实社交活动，不能写“搭子”和活动。
- 这种语气更适合年轻人旅居、小院、社区型民宿，不适合所有民宿。
- 老板人格化不能变成自夸，要靠具体行为支撑。

可转化为 Prompt 的规则：

- 民宿主可以像招呼朋友，但不要客服化。
- 用具体行为表达“服务好”，例如准备小甜品、每天打扫、整理院子。
- 没有活动时，不写“搭子”，可改成“适合自己慢慢待一会儿”。
- 对云朵情书样例，可转成：这是一个回到大理后能慢下来的小院，有窗边、院子、干净房间和晚上的小甜品。

### 4.0.5 已采样本 5：大理旅居 45 天租房指南

样本文件：

```text
samples/xhs/20260709-dali-rental-guide-005.json
```

账号类型：旅居攻略/个人经验账号。

公开互动：点赞 66，收藏 45，评论 74。

这条不是民宿主自营口吻，也不适合作为核心 few-shot。但它有一个重要价值：评论多，说明用户会被“清楚回答实际决策问题”的内容带动咨询。

可学习点：

- 标题用“时间跨度 + 指南/盘点”降低理解成本。
- 正文按区域、价格、优缺点、适合谁组织，信息很清楚。
- 适度写缺点和适合边界，会比只写优点更可信。
- 评论区集中问价格、怎么租、哪里租，说明“实用信息”能带来转化问题。

不能照搬的点：

- 它是旅居者攻略，不是民宿老板口吻。
- 租金、月租、区域、交通、生活成本都不能由模型编造。
- OTA 好评转小红书笔记时，不要强行写成“租房指南”。

可转化为 Prompt 的规则：

- 当老板明确提供月租、长住、价格、区域、交通信息时，可以考虑攻略式结构。
- 当只有 OTA 好评时，只借鉴它的清晰表达：真实细节、对住客的意义、适合边界。
- 对云朵情书样例，可借鉴的是“东门附近出行相对省心、外卖快、院子/房间细节适合慢一点住”，不要写价格和月租。

### 4.0.6 已采样本 6：民宿主理人内容方法

样本文件：

```text
samples/xhs/20260709-homestay-owner-content-method-006.json
```

账号类型：内容运营教学/实体店方法账号。

公开互动：点赞 499，收藏 594，评论 68。

这条不是民宿实际发布样本，但对我们现在的 Prompt 很重要。它解决的是“民宿主应该从什么角度写”，而不是“某一家民宿卖点是什么”。

可学习点：

- 民宿自营账号可以从一个真实经营瞬间切入，而不是从感谢客人或营销反问切入。
- 老板的情绪要落在具体行为上：整理房间、换床品、打理院子、回复消息、准备小甜品。
- 好内容不是喊“服务好”，而是让读者看到服务发生在什么细节里。
- 文案可以像日记，但必须让潜在住客感受到“这里有人认真经营”。
- 评论区出现民宿老板求指导，说明这种内容方法本身符合目标用户痛点。

不能照搬的点：

- 它是教学内容，不是最终发布文案。
- “满房”“回头客”“早餐”“院子”“下雨”“客人原话”等都必须有输入依据。
- 不能把“一天拍一条”变成我们 MVP 对老板的要求。

可转化为 Prompt 的规则：

- 开头优先写一个被看见的经营细节，而不是“作为主理人收到好评”。
- 如果好评说老板服务好，要转成具体行为，例如打扫房间、准备小甜品、整理院子。
- 正文可以有一点日常感，但不能脱离 OTA 好评和图片。
- 对云朵情书样例，可转成：“今天想把住客提到的几个小细节放出来，不是为了把这里夸得多特别，而是这些日常确实会影响住起来舒不舒服。”

### 4.0.7 已采样本 7：从此多了一个新身份

样本文件：

```text
samples/xhs/20260709-dali-owner-new-identity-yard-007.json
```

账号类型：民宿自营/主理人账号。

公开互动：点赞 1427，收藏 612，评论 352。

这是目前最核心的民宿主理人样本之一。它不是用“好评总结”写内容，而是用“我为什么做这家小院 + 这个空间有哪些真实细节”打动读者。

可学习点：

- 标题从身份变化切入：“从此多了一个新身份”，比“民宿推荐”更有故事感。
- 开头先讲主理人愿望：想拥有小院、逃离城市、慢慢生活。
- 正文用具体空间细节支撑生活方式：房间、客厅、商店、稻田落地窗、窗前望炉、草海、傍晚篝火、旅行故事画框。
- 结尾是轻邀请：“如果这也是你想要的生活”，不是硬销售。
- 评论区大量问招聘、床位、商店、宠物、过年空房，说明主理人故事能带来真实咨询。

不能照搬的点：

- 不能在没有输入依据时编造主理人创业故事、历时一年、房间数量、商店、稻田、篝火、草海。
- 这类写法适合有明确主理人故事和空间美学的民宿，不适合所有 OTA 好评生成。
- 如果只有一条 OTA 好评，不能自动写成老板人生故事。

可转化为 Prompt 的规则：

- 如果民宿档案提供了主理人故事，可以用“我们为什么做这家小院”作为开头。
- 如果没有主理人故事，只能用住客好评里的真实细节切入。
- 正文要把空间细节和生活方式连接起来，不要只写配置清单。
- 对云朵情书样例，不能写创业故事，但可以借鉴“轻邀请 + 小院生活感”：把东门位置、院子、房间打扫、小甜品串成住起来的节奏。

- 如果老板明确提供房型、空房日期、价格、长住/短住规则，可以生成更直接的转化型笔记。
- 如果只有 OTA 好评，只借鉴它的信息结构：在哪、住起来方便在哪里、哪些细节让人省心、适合谁。
- 对云朵情书样例，可以学习“位置和生活便利要说清楚”，但不能写房型、价格、日期和可住人数。

```text
samples/xhs/20260709-dali-guest-loved-yard-service-009.json
```

这是很适合我们 OTA 好评转笔记的样本。它不是民宿主口吻，但它把“房东服务好”拆成了很多真实动作。

不能照搬的点：

- 民宿主不能写“我订了这家”“老板让我感觉被爱”这类游客口吻。
- 房东父子、端午节、粽子、家常菜、资生堂用品、车位等都必须有输入依据。
- 民宿主版本不能自夸“我们让客人感到被爱”，要改成克制记录具体服务。

可转化为 Prompt 的规则：

- 当好评里出现“老板很好/服务好”时，优先寻找具体动作，而不是直接复述形容词。
- 可以把住客反馈整理成入住时间线：到达前、到达时、住下后、晚上回来。
- 对云朵情书样例，可转成：住客提到位置、外卖、院子、房间打扫、小甜品，这些都属于住起来会被感受到的小事。

### 4.1 大理米糖的院子

状态：待补充链接或截图。

来源线索：

- 用户指定为可参考民宿样本。
- 外部搜索引擎暂未稳定检索到精确结果。
- 小红书站内可能有内容，但自动读取不稳定，建议用链接或截图方式补充。

需要采集的内容：

- 小红书主页链接或笔记链接
- 3-5 篇代表性笔记
- 每篇标题、正文、标签
- 首图和组图结构
- 点赞、收藏、评论等公开数据
- 评论区是否有人问房价、位置、房型、预订方式

预期可学习点：

- 如果账号是民宿自营，可重点学习“老板/店铺口吻”如何写得不像感谢信。
- 如果内容围绕院子、房间、古城位置、生活方式展开，可拆成 few-shot 示例。
- 如果图片统一且有风格，可总结“民宿发布预览”的真实图片顺序。

需要特别判断：

- 是否依赖漂亮图片而不是文案本身。
- 是否存在无法复用的网红/达人表达。
- 是否有夸张承诺、价格引流或平台高风险表达。

## 5. 第一批通用样本拆解方向

### 5.1 院子型民宿

重点看：

- 标题是否把“院子”变成出行理由。
- 开头是否进入“来大理想慢下来/想住得舒服”的场景。
- 正文是否写清楚院子能做什么：喝茶、发呆、吹风、拍照、聊天。
- 图片是否先放院子，再放房间，再放周边。

可学习句式：

- “来大理，如果你不想把行程排太满，可以留一点时间给院子。”
- “挑民宿时，我会先看公共空间是不是能真的坐下来。”
- “这类小院更适合想慢慢逛、慢慢住的人。”

风险：

- 没有提供院子照片时，不要写成确定事实。
- 不要把“安静”自动写入，除非好评或资料明确提到。

### 5.2 古城位置型民宿

重点看：

- 标题是否强调吃逛方便，但不写具体距离。
- 正文是否避免“出门就是所有景点”这类夸大。
- 标签是否覆盖城市、古城、住宿、民宿推荐。

可学习句式：

- “想住在古城里，吃逛方便会省很多心。”
- “这条评价里比较有参考价值的是：出门找吃的方便。”
- “不想每天在路上折腾，可以优先看这种位置。”

风险：

- 没有具体距离时，不写“步行几分钟”。
- 没有交通信息时，不写“打车方便/停车方便”。

### 5.3 好评佐证型民宿

重点看：

- 好评是作为证据出现，而不是全文复述。
- 不用“客人说：”把正文写成评价搬运。
- 好评中的关键词转成卖点段落。

可学习句式：

- “这条评价里，比较有参考价值的是几个很具体的细节。”
- “比起空泛地说舒服，住客提到的这些点更能帮助判断。”
- “如果你也在意这些细节，发布前可以对照图片看。”

风险：

- 不要写“很多客人都说”，除非输入多条好评。
- 不要把个案服务写成长期承诺。
