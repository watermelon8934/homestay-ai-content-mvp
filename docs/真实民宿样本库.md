# 真实民宿样本库

## 1. 用途

这份文档用于沉淀真实民宿的小红书内容样本，帮助我们从“盲调 prompt”转向“基于好样本学习”。

每个样本不只是收集文案，而是拆解：

- 标题怎么吸引目标客人
- 开头怎么进入住宿选择场景
- 正文如何组织卖点
- 标签如何覆盖搜索词
- 图片如何承担种草作用
- 评论区是否出现咨询信号
- 哪些表达可以复用，哪些表达有风险

## 2. 采集优先级

优先采集：

- 真实民宿自营账号
- 大理、丽江、杭州、厦门、莫干山等民宿密集城市
- 有院子、古城、亲子、情侣、宠物、景观等明确场景的民宿
- 收藏、评论或咨询明显高于同类的笔记
- 评论区有人问位置、价格、房型、交通、预订方式

暂不优先：

- 纯达人广告
- 只有情绪没有住宿信息的内容
- 大量夸张词、极限词、低价噱头
- 无法被小体量民宿老板复用的内容

## 4. 待采集目标样本

### 4.1 大理米糖的院子

状态：待补充链接或截图。

来源线索：

- 用户指定为可参考民宿样本。
- 外部搜索引擎暂未稳定检索到精确结果。
- 小红书站内可能有内容，但自动读取不稳定，建议用链接或截图方式补充。

需要采集的内容：

- 小红书主页链接或笔记链接
- 3-5 篇代表性笔记
- 每篇标题、正文、标签
- 首图和组图结构
- 点赞、收藏、评论等公开数据
- 评论区是否有人问房价、位置、房型、预订方式

预期可学习点：

- 如果账号是民宿自营，可重点学习“老板/店铺口吻”如何写得不像感谢信。
- 如果内容围绕院子、房间、古城位置、生活方式展开，可拆成 few-shot 示例。
- 如果图片统一且有风格，可总结“民宿发布预览”的真实图片顺序。

需要特别判断：

- 是否依赖漂亮图片而不是文案本身。
- 是否存在无法复用的网红/达人表达。
- 是否有夸张承诺、价格引流或平台高风险表达。

## 5. 第一批通用样本拆解方向

### 5.1 院子型民宿

重点看：

- 标题是否把“院子”变成出行理由。
- 开头是否进入“来大理想慢下来/想住得舒服”的场景。
- 正文是否写清楚院子能做什么：喝茶、发呆、吹风、拍照、聊天。
- 图片是否先放院子，再放房间，再放周边。

可学习句式：

- “来大理，如果你不想把行程排太满，可以留一点时间给院子。”
- “挑民宿时，我会先看公共空间是不是能真的坐下来。”
- “这类小院更适合想慢慢逛、慢慢住的人。”

风险：

- 没有提供院子照片时，不要写成确定事实。
- 不要把“安静”自动写入，除非好评或资料明确提到。

### 5.2 古城位置型民宿

重点看：

- 标题是否强调吃逛方便，但不写具体距离。
- 正文是否避免“出门就是所有景点”这类夸大。
- 标签是否覆盖城市、古城、住宿、民宿推荐。

可学习句式：

- “想住在古城里，吃逛方便会省很多心。”
- “这条评价里比较有参考价值的是：出门找吃的方便。”
- “不想每天在路上折腾，可以优先看这种位置。”

风险：

- 没有具体距离时，不写“步行几分钟”。
- 没有交通信息时，不写“打车方便/停车方便”。

### 5.3 好评佐证型民宿

重点看：

- 好评是作为证据出现，而不是全文复述。
- 不用“客人说：”把正文写成评价搬运。
- 好评中的关键词转成卖点段落。

可学习句式：

- “这条评价里，比较有参考价值的是几个很具体的细节。”
- “比起空泛地说舒服，住客提到的这些点更能帮助判断。”
- “如果你也在意这些细节，发布前可以对照图片看。”

风险：

- 不要写“很多客人都说”，除非输入多条好评。
- 不要把个案服务写成长期承诺。
