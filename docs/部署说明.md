# 部署说明

## 当前工程边界

MVP 当前拆成两层：

- 前端：`app/`，普通 Vite + React 静态前端。
- 后端：后续用 Java / Spring Boot 实现。

前端只负责：

- 填写民宿资料
- 粘贴 OTA 好评
- 上传图片
- 展示标题、正文、标签、配图建议
- 复制发布内容

后端只需要实现一个接口：

```text
POST /api/generate-note
```

## 前端部署

推荐先用 Vercel 部署前端。

GitHub 仓库：

```text
https://github.com/watermelon8934/homestay-ai-content-mvp
```

Vercel 导入项目时设置：

```text
Root Directory: app
Framework Preset: Vite
Build Command: npm run build
Output Directory: dist
```

如果 Java 后端还没上线，前端可以先不配置 `VITE_API_BASE_URL`，生成会自动走本地示例内容，适合演示流程。

Java 后端上线后，再配置：

```text
VITE_API_BASE_URL=https://你的后端域名
```

## Java 后端接口约定

请求：

```json
{
  "review": "OTA 好评文本",
  "property": {
    "name": "民宿名称",
    "city": "城市",
    "roomCount": "",
    "highlights": "",
    "surroundings": "",
    "petPolicy": "",
    "notes": ""
  }
}
```

响应：

```json
{
  "ok": true,
  "draft": {
    "id": "唯一 ID",
    "createdAt": 1760000000000,
    "property": {},
    "reviewInput": "原始好评",
    "titles": ["标题 1", "标题 2"],
    "body": "正文",
    "tags": ["#大理民宿"],
    "imageIdeas": ["配图建议"],
    "rationale": "生成依据",
    "risks": ["风险提醒"]
  }
}
```

失败：

```json
{
  "ok": false,
  "error": "错误说明"
}
```

## 密钥规则

DeepSeek / OpenAI 等模型 API Key 只放 Java 后端。

不要放在：

- 前端代码
- GitHub 仓库
- `VITE_` 开头的环境变量

## 本地清理规则

这些目录都是本地缓存或构建产物，可以删除后重新生成：

```text
app/node_modules/
app/dist/
app/.output/
app/.wrangler/
app/.tanstack/
app/storage/
```

这些文件需要保留：

```text
app/package.json
app/src/
app/index.html
app/vite.config.ts
```
