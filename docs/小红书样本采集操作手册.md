# 小红书样本采集操作手册

## 目的

这个采集流程不是为了大规模抓取内容，而是为了给「民宿 AI 内容增长工具」建立一批可学习的真实样本。

第一阶段目标：

- 找到 20-50 篇表现较好的民宿小红书笔记
- 拆解标题、开头、卖点、标签、图片结构
- 形成 few-shot 示例和质量评测标准
- 反哺生成 Prompt，让文案不再像生硬的感谢信

## 当前已安装的工具

项目里已经安装：

- Playwright：免费，优先使用，适合当前验证
- Crawlee：免费，更像正式爬虫框架，但对本机权限要求更高
- Firecrawl 脚本：已预留，但需要 API Key，当前不作为主线

采集小红书搜索结果：

## 关键词策略

第一轮不要追求多，先围绕真实民宿老板可能关心的场景采样。

### 品牌词

用于观察具体民宿账号或用户笔记：

```text
大理米糖的院子
米糖的院子
大理古城 米糖的院子
```

### 城市加品类词

用于观察大盘表达方式：

```text
大理民宿
大理古城民宿
大理洱海民宿
大理院子民宿
```

### 人群场景词

用于学习「适合谁」的写法：

```text
大理亲子民宿
大理情侣民宿
大理闺蜜民宿
大理一个人住的民宿
```

### 决策痛点词

用于学习小红书上更像真实搜索需求的标题：

```text
大理民宿怎么选
大理古城住宿推荐
大理住哪里方便
大理古城附近住宿
大理民宿避雷
```

## 采样数量

建议第一轮这样跑：

- 每个关键词抓 8-12 条
- 先跑 8-10 个关键词
- 得到约 80-120 条搜索结果
- 人工筛掉弱相关、广告感太强、标题太空的内容
- 最终保留 20-50 条作为高质量样本

## 判断好样本的标准

优先保留：

- 点赞、收藏、评论相对更高的笔记
- 标题像真实用户会点开的内容
- 开头直接进入住宿选择场景
- 卖点具体，不是泛泛而谈
- 图片结构清晰，比如院子、房间、窗景、公共区、周边
- 评论区有人问价格、位置、预订、适合几人住

谨慎使用：

- 过度夸张的标题
- 明显搬运或营销号内容
- 和民宿实际经营弱相关的旅行攻略
- 只靠情绪词，没有具体卖点的内容

## 拆解维度

每条样本建议补充这些字段：

```text
标题结构：
开头方式：
核心卖点：
标签：
图片结构：
评论区咨询信号：
可学习点：
风险点：
是否适合作为 few-shot：
```

## 对我们产品的作用

这些样本最终会变成三类资产：

1. 生成示例：告诉模型什么叫“像小红书民宿笔记”
2. 评测标准：判断生成内容是否自然、具体、可信
3. 内容雷达雏形：未来根据公开反馈反哺生成策略

第一版先人工筛样本，不做大规模自动采集。这样更稳，也更符合 MVP 阶段的节奏。
